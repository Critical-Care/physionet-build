Author Guidelines


Process Overview

Once a version of a project is published, it falls into the public domain and cannot be changed. Contributors will have no control over it. They may however edit their project and publish subsequent versions.


Project Creation

An account is required for creating projects. The project creation link can be found in the navbar under the publish panel, or your project home page, which lists all of your authoring projects.

Projects are not accessible by or even visible to anyone, except initially for the submitting author, and subsequently also for invited co-authors.


Authors

Each project can have one submitting author. who is the user who creates the project. Only the submitting author can invite, remove, and order co-authors, fill in the project metadata, upload/edit files, and submit the project for review/publication.

Co-authors are invited by email. Each co-author must have a PhysioNet account.


Uploading Files

All projects start with a moderately low size quota. You may request more storage if needed. Recall that only the submitting author may upload/edit files


Access Policies

Once published, the descriptive metadata of any project will be publically available. The following access policies control access to the files:

  1. Open: Anyone can access the files, as long as they conform to the terms of the license. No data use agreement (DUA) is provided.
  2. Restricted: Only logged in users who sign the specified DUA can access the files.
  3. Credentialed: Only PhysioNet credentialed users can access the files.

We strongly encourage using the 'open' access policy, and will reject non-open submissions that are not justified.


Review Process

If the project passes all of its automatic checks, the submitting author may submit the project for review/publication. Once all the co-authors approve the review submission, a set of reviewers is assigned to the project.
