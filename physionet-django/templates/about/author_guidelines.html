Author Guidelines


Process Overview

Once a version of a project is published, it falls into the public domain and cannot be changed. Contributors will have no control over it. They may however edit their project and publish subsequent versions.

It is important to separate the concepts of a project, and published versions of a project. A user may create a project and edit its fields. If it is submitted and accepted for publication, a version of the project is published, which falls into the public domain and cannot be edited. The original project


Project Creation

An account is required for creating projects. The project creation page can be found in the navbar under the publish panel, or your project home page, which lists all of your authoring projects.

Projects are not accessible by, or even visible to anyone, except initially for the submitting author, and subsequently also for invited co-authors.

When creating a project, choose the correct resource type, as this cannot be changed later. If you would like to contribute both data and code from a single related project, submit them in separate projects, and refer them to each other.


Authors

Each project can have one submitting author, and many co-authors. The user who creates the project is the initial submitting author, though they can assign this role to another co-author if they want. Only the submitting author can invite, remove, and *order co-authors, fill in the project metadata, upload/edit files, and submit the project for review/publication. *The author order is the order in which author names will be displayed in the published project.

The submitting author can invite a co-author to the project by specifying an email address. An email will be sent to the target email address. If a PhysioNet account is already associated with the specified email, the invitation will appear in its project invitations page. If not, once they associate the email to their existing account, or create a new account with the email, they will be able to view and accept the invitation in the specified location.

Co-authors must fill in their own affiliations. Co-authors can view the project content as it is being prepared, but are unable to make any edits to the files and descriptive metadata.


Access Policies

Once published, the descriptive metadata of any project will be publically available. The following access policies control access to the files:

  1. Open: Anyone can access the files, as long as they conform to the terms of the license. No data use agreement (DUA) is provided.
  2. Restricted: Only logged in users who sign the specified DUA can access the files.
  3. Credentialed: Only PhysioNet credentialed users can access the files.

We strongly encourage using the 'open' access policy, and will reject non-open submissions that are not justified.


Uploading Files

Recall that only the submitting author can upload/edit files. All projects start with a moderately low size quota. The submitting author may request more storage if needed.


Review Process

If the project passes all of its automatic checks, the submitting author may submit the project for review/publication. Once all the co-authors approve the review submission, a set of reviewers is assigned to the project.

Once all the reviewers have completed their review, the editor will make the decision to accept or reject the publication.
